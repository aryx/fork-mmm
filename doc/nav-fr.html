MMM : navigation

Liens hypertextes et objets actifs

Dans un document HTML affiché, les liens hypertextes (ou ancres) sont habituellement affichés en bleu souligné. L'auteur du document peut avoir indiqué d'autres couleurs, mais dans tous les cas:
  • le pointeur de la souris devient une "main"
  • une adresse de destination s'affiche en bas de la fenêtre
Il existe également des objects actifs, comme certaines images; certaines sont des simples boutons, et on voit la même choses que pour les ancres dans le texte. D'autres contiennent plusieurs zones actives : le pointeur de souris est alors une flêche qui pointe vers le haut et la gauche. Soit l'image est n'est pas encore chargée, et alors en cliquant avec le bouton 1 on fait apparaître un menu des liens possibles, soit l'image est déjà chargée, et il suffit de cliquer à l'endroit voulu.

Il existe différentes manières d'activer un ancre, résumées ci-dessous:

Bouton-1
affiche le document pointé par ce lien dans la même fenêtre, sauf indication spécifique de l'auteur du document (frames)
Shift + Bouton 1
sauve le document pointé par le lien (après demande d'un nom de fichier par une boîte de dialogue)
Bouton 3
affiche le document pointé par ce lien dans une nouvelle fenêtre
Control + Bouton 1
affiche un menu d'activation du lien, pouvant contenir d'autres fonctions de navigation
Le choix particulier des boutons souris et touches supplémentaires est réglable par l'utilisateur en éditant le fichier de préférence, et en modifiant les ressources adéquates.

Barre de commande

La barre de commande propose les boutons de navigation suivants:
Home
qui va au document Home, tel que défini ici
Précédent
qui revient au document précédemment affiché (ne marche pas avec les frames)
Suivant
qui revient au document suivant (ne marche pas avec les frames)
Ces actions sont également disponibles à partir du menu Navigation, ainsi que le déplacement dans l'historique de navigation. Les raccourcis du menu sont réglables par des ressources dans le fichier de préférences.

Autres éléments d'interface

Recharger, Mettre à jour

Recharger provoque le rechargement complet du document à partir de son serveur d'origine, en indiquant au proxy qu'il ne doit pas consulter son cache pour répondre.

Mettre à jour demande au serveur si le document a été modifié depuis qu'il a été affiché dans MMM (en particulier, si MMM a initialement obtenu le document dans le cache du proxy, ce menu permet de vérifier auprès du serveur que le document est à jour). Certains serveurs ne savent pas répondre à ce type de requête (pourtant standard en HTTP/1.0), et renverront systématiquement une nouvelle copie du document, même identique.

Documents HTML

En-têtes

Le titre du document est également un menu qui affiche les en-têtes HTTP de ce document. Certaines en-têtes sont activables (p.ex. Refresh).

Smooth scroll

L'icone avec deux petites flêches change le mode de défilement du document. En présence de grosses tables ou images (ie. dont la taille dépasse la taille de la fenêtre MMM), le défilement ne fonctionne pas correctement, et il faut alors passer en mode smooth scroll pour voir l'intégralité du document. Ce mode peut être activé par défaut dans les préférences, mais il est couteux.

Links

A droite de l'icône, le bouton Links, si il est actif, donne accès au menu qui contient les liens particuliers éventuellement définis par l'auteur du document. Dans le cas où l'affichage des frames n'a pas été activé, ce menu contient en particulier une entrée pour chaque frame du document.

Recherche

En tapant Control-S dans la fenêtre d'affichage, on accède à une boîte de dialogue permettant de chercher du texte dans le document affiché (ne cherche pas dans les cellulles des tables, ou dans les frames).

Barre de progrès

En bas d'une fenêtre MMM, une barre de progrès permet de suivre l'évolution du chargement du document demandé. De plus, si la barre devient rouge, cela signifie qu'une erreur de syntaxe a été détectée dans le document HTML (et donc, éventuellement, que l'affichage pourra présenter des défauts).

Affichage du source

Le menu Voir le source ouvre une fenêtre sur le source HTML du document affiché. Ce source peut être décoré en couleurs (voir comment). On peut également voir les positions des erreurs éventuelles de syntaxe dans le HTML; les erreurs apparaissent soulignées, et le message correspondant est affiché lorsque la souris passe sur le lieu de l'erreur. Il est possible d'éditer directement le source du document, et de réafficher la version modifiée; dans le cas où le document a été accédé par une URL file:, on peut également sauver la version modifiée.
